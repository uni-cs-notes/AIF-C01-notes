# 📚 AWS Certified AI Practitioner (AIF-C01) Documentation Structure




### 📂 01-ML-Fundamentals
**Description:** This domain establishes the "Why" and "How" of Machine Learning. It covers the difference between AI, ML, and Deep Learning, as well as the standard lifecycle of preparing data, training models, and evaluating performance.
* **1.1-Concepts.md:** Definitions of AI vs. ML vs. DL.
* **1.2-ML-Lifecycle.md:** Detailed breakdown of the data-to-deployment pipeline.
* **1.3-SageMaker-Foundations.md:** Introduction to the SageMaker ecosystem.

### 📂 02-Generative-AI
**Description:** Focuses specifically on modern GenAI technologies. You will document Foundation Models (FMs), Large Language Models (LLMs), and how to optimize them using prompt engineering and fine-tuning.
* **2.1-Intro-to-GenAI.md:** What are FMs and LLMs?
* **2.2-Amazon-Bedrock.md:** Using Bedrock to access third-party models via API.
* **2.3-Prompt-Engineering.md:** Best practices for Zero-shot, Few-shot, and Chain-of-thought prompting.

### 📂 03-AWS-AI-Services
**Description:** A guide to AWS's "ready-made" AI tools. These services allow developers to add intelligence (vision, speech, text) to applications without managing underlying infrastructure or models.
* **3.1-Vision-Services.md:** Amazon Rekognition and Textract.
* **3.2-Language-Services.md:** Amazon Comprehend, Translate, and Lex.
* **3.3-Speech-Services.md:** Amazon Polly and Transcribe.

### 📂 04-Responsible-AI-Security
**Description:** Covers the ethical and safety guardrails required in cloud AI. This includes managing bias, ensuring explainability, and understanding the Shared Responsibility Model between the user and AWS.
* **4.1-Responsible-AI.md:** Bias detection, fairness, and SageMaker Clarify.
* **4.2-Security-Compliance.md:** IAM roles, encryption, and AWS CloudTrail for auditing.
* **4.3-Shared-Responsibility.md:** Defining who secures what in the AI stack.
